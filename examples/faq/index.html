Frequently asked questions

What is PresetCSS? −
PresetCSS is a utility CSS generator driven by a single flat TOML file. You declare your design tokens (colors, spacing, fonts, etc.) and get an optimized .css file back. No JavaScript, no plugins, no Node.js runtime required.
How is this different from Tailwind CSS? +
Tailwind uses a tailwind.config.js JavaScript file and relies on PostCSS. PresetCSS uses a plain preset.toml and is a compiled Rust binary. Tailwind has a much larger class vocabulary; PresetCSS is intentionally minimal — you define exactly what you need and nothing else.
Do I need Rust installed to use PresetCSS? +
Only to install via cargo install presetcss. Pre-built binaries for Linux, macOS, and Windows are planned for future releases, which will require no Rust installation. The browser playground works fully offline via WASM.
What languages / frameworks does it work with? +
Any project that can link a .css file. PHP, Ruby, Django, Laravel, plain HTML — it doesn't matter. The only integration point is a <link rel="stylesheet"> tag.
How does tree-shaking work? +
Add a content glob to your preset:

content = ["src/**/*.html", "templates/**/*.html"]
PresetCSS scans those files with an Aho-Corasick automaton and only emits classes that actually appear in the templates. Near-miss typos like bg-primry get a friendly warning.
Can I use hover, focus, and dark-mode variants? +
Yes — via the [variants] section. Variants are opt-in per token group, so you only pay for what you use. Supported variants: hover, focus, focus-visible, active, disabled, dark, responsive, and more.
Is there a VS Code extension? +
Not yet, but it's on the roadmap. For now, the explain command provides class → CSS rule lookup, and watch gives instant feedback in the browser.
